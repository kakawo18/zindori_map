# ジンドリマップ (Web MVP)

現実世界を歩いた GPS 軌跡から領土（ポリゴン）を生成し、面積・形・新規性に応じてポイントを獲得する
個人記録型の位置情報ウォーキングゲーム。`仕様.txt` の MVP 仕様を Web アプリとして実装したもの。

## 起動

```bash
npm install
npm run dev      # http://localhost:5173
npm test         # コアロジックのテスト
npm run build
```

Geolocation API は **localhost または HTTPS でのみ動作**する。
実機スマホで検証する場合は `npm run dev -- --host` と HTTPS トンネル（ngrok / cloudflared 等）を併用する。

## 構成

| ディレクトリ | 役割 |
| --- | --- |
| `src/core/` | 純粋なゲームロジック（UI・永続化に非依存）。テスト対象はここ。 |
| `src/data/` | 永続化層。`GameRepository` インターフェース + IndexedDB 実装。 |
| `src/game/` | 実績・ミッション・ランキングのルール。 |
| `src/hooks/` | `useTracker`（測位）、`useGame`（状態と確定処理）。 |
| `src/components/` | 画面。 |
| `src/config.ts` | ゲームバランス定数。仕様書で「要実測調整」の値はすべてここ。 |

### 確定処理のパイプライン

`src/core/finalize.ts` が仕様のコアループを 1 本にまとめている。

1. `antiCheat.ts` — 時速 20km 超の区間を除外
2. `simplify.ts` — Douglas-Peucker で GPS ノイズ除去
3. `loop.ts` — 始点・終点が 20m 以内ならループ成立、Shoelace 公式で面積算出
4. `h3cells.ts` — ポリゴンを H3 セル（res12）に変換し、既取得セルを差し引く
5. `scoring.ts` — 段階式の面積ボーナス × Polsby-Popper のコンパクトさボーナス。未成立時は距離ベースの救済ポイント

## 仕様との対応

実装済み:

- フォアグラウンド限定トラッキング（タブ非表示で自動停止）
- GPS 途切れ 30 秒で「記録一時停止」、直線補間せず区間を分割
- H3 セル単位の未取得→取得済み管理（再訪では面積ポイント加算なし）
- 段階式面積ボーナス / Polsby-Popper コンパクトさボーナス / 救済ポイント
- 異常速度検知による区間除外
- 開始時の安全警告ダイアログ
- デイリーランキング、カウントベースの実績、1 日 1 つの自動ミッション
- 領土は端末内のみ保存・非公開

MVP 対象外（仕様どおり未実装）: 領土の奪い合い、都道府県制覇実績、フレンド機能、
高度な不正対策、バックグラウンド常時トラッキング、マネタイズ。

## 既知の制約 / 次の一手

- **バックエンド未接続**: 全データは IndexedDB に閉じている。`GameRepository` を Supabase(PostGIS) 実装に差し替える想定。
- **ランキングのライバルはダミー**: 日付シードから決定論的に生成している。サーバー集計に差し替えが必要。
- **オフラインキューは積むだけ**: `outbox` ストアに溜まるが送信先がないため flush しない。
- **地図タイル**: OpenStreetMap の公式タイルを直接参照している。本番では利用規約上、自前ホスト or 商用タイルプロバイダへの差し替えが必須。
- **プライバシーポリシーは草案**: 「情報」タブの文面はリリース前に正式版へ差し替えること。
